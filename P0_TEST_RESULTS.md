### 测试 3: 选择器生成策略 ✅

**测试场景**: 分析 GitHub 首页的选择器质量

**选择器统计** (20 个样本):
```
data-* 属性:  0 个 (0.0%)   - 最稳定
ID 选择器:    1 个 (5.0%)   - 次稳定
Class 选择器: 6 个 (30.0%)  - 一般
文本选择器:   5 个 (25.0%)  - 一般
属性选择器:   0 个 (0.0%)   - 较差
路径选择器:   8 个 (40.0%)  - 最差
```

**选择器质量评分**: 2.25/5.0

**示例选择器**:
```
1. #hero-section-brand-heading
   └─ 策略: ID 选择器（稳定）

2. div:nth-child(1) > section:nth-child(1) > div:nth-child(1) > div:nth-child(6) > h2:nth-child(1)
   └─ 策略: nth-child 路径（较差，但唯一）

3. h2.Primer_Brand__Heading-module__Heading___IVpmp.Primer_Brand__Heading-module__Heading-font--mona-sans___SCnTx.Primer_Brand__Heading-module__Heading--subhead-large____C6GZ
   └─ 策略: Class 选择器（一般，但 class 名很长）

4. a.px-2.tmp-py-4.color-bg-accent-emphasis
   └─ 策略: Class 选择器（一般）

5. button.close-button.color-fg-on-emphasis
   └─ 策略: Class 选择器（一般）
```

**分析**:
- ✅ 多策略生成机制正常工作
- ✅ 优先使用 ID 和 Class（35%）
- ⚠️ 路径选择器占比较高（40%）
- ⚠️ GitHub 页面缺少 data-* 属性

**原因分析**:
1. GitHub 首页是现代 React 应用
2. 大量使用动态生成的 class 名（如 `Primer_Brand__Heading-module__Heading___IVpmp`）
3. 缺少 data-testid 等测试属性
4. 很多元素没有稳定的 ID 或 class

**改进空间**:
- 可以添加更智能的 class 过滤（识别 CSS Modules 模式）
- 可以优化路径选择器的生成（缩短路径）
- 可以添加相对定位策略（基于文本内容）

**验证点**:
- ✅ 6 层策略都在工作
- ✅ 选择器生成不会失败
- ✅ 优先级顺序正确
- ⚠️ 质量评分偏低（但符合预期）

**结论**: ✅ **P0-3 修复有效，质量可接受**
