### 3.3 Step 2: 简化 parse_json_response

当前结构（4 层 fallback，大括号匹配重复 3 次）：

```
Level 1: _try_parse() 内部函数
  ├── Markdown code block 提取
  ├── 直接 JSON parse
  ├── 大括号匹配提取        ← 第 1 次
  └── 正则 fallback
Level 2: 双大括号标准化 ({{ → {)
Level 3: Markdown 提取      ← 重复 Level 1
Level 4: 大括号匹配         ← 第 2 次重复
Level 5: 正则搜索           ← 第 3 次重复
```

**重构为单一管道**:

```python
def parse_json_response(self, content: str) -> Optional[dict]:
    candidates = [content, content.replace("{{", "{").replace("}}", "}")]
    for text in candidates:
        result = self._try_parse(text)
        if result is not None:
            return result
    return None
```

保留 `_try_parse` 内部的 3 阶段管道（code-fence → balanced-brace → regex），删除外层重复的 Level 3-5。

### 3.4 Step 3（低优先级）: 文档化 `response_format` 使用

`json_mode=True` 时已传递 `response_format={"type": "json_object"}`（line 485）。在新代码中优先使用此路径，减少对 `parse_json_response` 的依赖。

async def run(self, task, start_url=None, max_steps=8):
    # 1. 初始化
    error = await self._initialize_session(task, start_url, max_steps)
    if error:
        return error

    # 2. 推断意图
    task_intent = await self._infer_task_intent(task)

    # 3. 主循环
    for step_no in range(1, max_steps + 1):
        result = await self._execute_step(step_no, task, task_intent, ...)
        if result.should_return:
            return result.return_value

    # 4. 兜底结果
    return await self._build_final_result(task, task_intent, steps, data)
