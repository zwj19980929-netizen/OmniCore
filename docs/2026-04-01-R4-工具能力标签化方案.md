# R4: 工具能力标签化方案

> 优先级：P1 | 预估工作量：1 天 | 依赖：无 | 被依赖：无（但为未来并发优化打基础）

---

## 1. 目标

当前工具输出类型判断依赖 `"web" in lower_tool` 这种字符串 heuristic，脆弱易误判。工具无并发安全标记、无资源需求声明，调度器只能靠 `serialized` 和 `max_parallelism` 硬编码来控制并发。新增工具（尤其是 MCP 工具）时，调度器完全不知道它的特性。

本方案给 `ToolSpec` 增加结构化能力标签，让调度器基于声明式元数据做决策。

**验收标准**：

- `ToolSpec` 新增 `output_type`、`concurrent_safe`、`needs_network`、`destructive` 字段
- 所有 10 个内置工具 + MCP 工具有结构化能力声明
- `_extract_typed_output()` 改为读取 `ToolSpec.output_type`，零字符串 heuristic
- `_select_batch_indexes()` 利用 `concurrent_safe` 标记决定并行批次
- 全量测试通过

## 2. 现有代码基础

| 组件 | 文件 | 当前状态 |
|------|------|---------|
| ToolSpec 定义 | `core/tool_protocol.py` | 字段：name, task_type, description, risk_level, tags, input_schema |
| RegisteredTool | `core/tool_registry.py` | 额外字段：adapter_name, max_parallelism, serialized, plugin_id |
| 内置工具注册 | `core/tool_registry.py` `_register_builtin_tools()` | 10 个工具硬编码注册 |
| 输出类型判断 | `core/task_executor.py` `_extract_typed_output()` | `"browser"/"web" in lower_tool` → TEXT_EXTRACTION；`"file" in lower_tool` → FILE_DOWNLOAD；`"terminal"/"system" in lower_tool` → COMMAND_OUTPUT |
| 并行批次选择 | `core/task_executor.py` `_select_batch_indexes()` | 基于 `registered_tool.serialized` + `max_parallelism` |

**关键发现**：

1. `_extract_typed_output()` 的 heuristic 会把 MCP 工具名如 `mcp.github.search_issues` 误判（不含 web/browser/file/terminal 关键词，直接 fallback 到 None）
2. `ToolSpec.tags` 字段已存在但仅用于搜索/展示，未被调度器消费
3. `RegisteredTool.serialized` 是布尔值，不够表达"可并发但不可与特定工具并行"的场景
4. MCP 工具注册时无任何能力声明，全部使用默认值（`max_parallelism=1`, `serialized=False`）

## 3. 架构设计

### 3.1 能力标签体系

```
ToolSpec 能力标签（新增字段）：
┌──────────────────────────────────────────┐
│  output_type: str                        │
│    "text_extraction"  — 网页/API 文本    │
│    "file_operation"   — 文件读写结果     │
│    "command_output"   — 命令行输出       │
│    "structured_data"  — JSON/表格数据    │
│    "binary"           — 二进制文件       │
│    "generic"          — 默认/未知        │
│                                          │
│  concurrent_safe: bool = True            │
│    True  — 可与其他工具并行              │
│    False — 必须独占执行                  │
│                                          │
│  needs_network: bool = False             │
│    True  — 需要网络访问                  │
│    False — 纯本地操作                    │
│                                          │
│  destructive: bool = False               │
│    True  — 会修改文件/系统/外部状态      │
│    False — 只读操作                      │
└──────────────────────────────────────────┘
```

### 3.2 内置工具能力声明

| 工具名 | output_type | concurrent_safe | needs_network | destructive |
|--------|------------|-----------------|---------------|-------------|
| `web.fetch_and_extract` | `text_extraction` | `True` | `True` | `False` |
| `web.smart_extract` | `text_extraction` | `True` | `True` | `False` |
| `browser.interact` | `text_extraction` | `False` | `True` | `True` |
| `file.read_write` | `file_operation` | `True` | `False` | `True` |
| `api.call` | `structured_data` | `True` | `True` | `True` |
| `system.control` | `command_output` | `False` | `False` | `True` |
| `terminal.execute` | `command_output` | `False` | `False` | `True` |
| `terminal.read_file` | `file_operation` | `True` | `False` | `False` |
| `terminal.edit_file` | `file_operation` | `False` | `False` | `True` |
| `terminal.search` | `text_extraction` | `True` | `False` | `False` |

## 4. 详细设计

### 4.1 修改 `core/tool_protocol.py` — 扩展 ToolSpec

```python
class ToolSpec:
    """工具规格定义"""

    def __init__(
        self,
        name: str,
        task_type: str,
        description: str,
        risk_level: str = "low",
        tags: list = None,
        input_schema: dict = None,
        # ── 新增能力标签 ──
        output_type: str = "generic",
        concurrent_safe: bool = True,
        needs_network: bool = False,
        destructive: bool = False,
    ):
        self.name = name
        self.task_type = task_type
        self.description = description
        self.risk_level = risk_level
        self.tags = tags or []
        self.input_schema = input_schema or {}
        # 新增
        self.output_type = output_type
        self.concurrent_safe = concurrent_safe
        self.needs_network = needs_network
        self.destructive = destructive
```

### 4.3 修改 MCP 工具注册 — 从配置推断能力

```python
# core/tool_registry.py _register_mcp_tools() 中

def _register_mcp_tools(self):
    # ... 现有 MCP 工具发现逻辑 ...

    for tool_info in mcp_tools:
        # 从 MCP Server 元数据或 mcp_servers.yaml 中读取能力声明
        capabilities = tool_info.get("capabilities", {})

        spec = ToolSpec(
            name=f"mcp.{server_name}.{tool_info['name']}",
            task_type="mcp_handler",
            description=tool_info.get("description", ""),
            risk_level=capabilities.get("risk_level", "medium"),
            output_type=capabilities.get("output_type", "generic"),
            concurrent_safe=capabilities.get("concurrent_safe", True),
            needs_network=capabilities.get("needs_network", True),  # MCP 默认需要网络
            destructive=capabilities.get("destructive", False),
        )
        self.register(RegisteredTool(spec=spec, adapter_name="mcp_handler"))
```

同时在 `config/mcp_servers.yaml` 中支持可选的能力声明：

```yaml
servers:
  github:
    command: "npx"
    args: ["-y", "@modelcontextprotocol/server-github"]
    enabled: true
    capabilities:          # 可选，覆盖该 server 下所有工具的默认值
      needs_network: true
      destructive: true    # GitHub API 有写操作
      output_type: "structured_data"
```

### 4.4 修改 `core/task_executor.py` — 基于 ToolSpec 判断输出类型

```python
# 原来的 _extract_typed_output:
#   lower_tool = tool_name.lower()
#   if "browser" in lower_tool or "web" in lower_tool:
#       output_type = TaskOutputType.TEXT_EXTRACTION
#   ...

# 改为：
from core.tool_registry import get_builtin_tool_registry

_OUTPUT_TYPE_MAP = {
    "text_extraction": TaskOutputType.TEXT_EXTRACTION,
    "file_operation": TaskOutputType.FILE_DOWNLOAD,
    "command_output": TaskOutputType.COMMAND_OUTPUT,
    "structured_data": TaskOutputType.TEXT_EXTRACTION,  # 结构化数据也按文本提取处理
    "binary": TaskOutputType.FILE_DOWNLOAD,
}


def _extract_typed_output(task: dict, outcome: dict) -> Optional[dict]:
    """基于 ToolSpec.output_type 提取类型化输出"""
    tool_name = task.get("tool_name", "")
    registry = get_builtin_tool_registry()
    registered = registry.get_by_name(tool_name)

    if registered and registered.spec.output_type != "generic":
        output_type = _OUTPUT_TYPE_MAP.get(
            registered.spec.output_type,
            TaskOutputType.TEXT_EXTRACTION,
        )
    else:
        # fallback: 对未注册工具仍用旧逻辑（兼容期）
        output_type = _guess_output_type_by_name(tool_name)

    return _build_typed_output(task, outcome, output_type)
```

### 4.5 修改 `_select_batch_indexes()` — 基于能力标签调度

```python
def _select_batch_indexes(state, ready_indexes):
    """基于工具能力标签选择并行批次"""
    registry = get_builtin_tool_registry()
    batch = []
    has_exclusive = False  # 是否已有非并发安全工具

    for idx in ready_indexes:
        task = state["task_queue"][idx]
        tool_name = task.get("tool_name", "")
        registered = registry.get_by_name(tool_name)

        if registered:
            concurrent_safe = registered.spec.concurrent_safe
            # 同时考虑 RegisteredTool.serialized（向后兼容）
            if registered.serialized:
                concurrent_safe = False
        else:
            concurrent_safe = True  # 未知工具默认可并发

        if not concurrent_safe:
            if not batch:
                # 独占工具，单独执行
                return [idx]
            else:
                # 已有其他任务在批次中，这个独占工具等下一批
                continue

        if has_exclusive:
            continue

        # 检查同工具并行度限制（原有逻辑保留）
        # ...

        batch.append(idx)

    return batch
```

## 5. 典型场景

### 场景：MCP GitHub 工具的输出类型自动识别

**改造前**：
```python
tool_name = "mcp.github.search_issues"
lower_tool = tool_name.lower()
# "browser" not in lower_tool → False
# "web" not in lower_tool → False
# "file" not in lower_tool → False
# "terminal" not in lower_tool → False
# → output_type = None（丢失类型化输出！）
```

**改造后**：
```python
tool_name = "mcp.github.search_issues"
registered = registry.get_by_name(tool_name)
# registered.spec.output_type = "structured_data"
# → output_type = TaskOutputType.TEXT_EXTRACTION
# → 正确提取结构化输出
```

## 7. 风险与注意事项

| 风险 | 缓解措施 |
|------|---------|
| ToolSpec 新增字段导致现有代码实例化报错 | 所有新字段有默认值，向后兼容 |
| MCP 工具能力声明缺失（yaml 中未配置） | 提供合理默认值（`concurrent_safe=True`, `needs_network=True`, `output_type="generic"`） |
| 插件工具（`@tool_plugin`）未适配能力标签 | 插件装饰器新增可选 capabilities 参数，未提供时使用默认值 |
| `_extract_typed_output` fallback 逻辑保留旧 heuristic | 在兼容期保留 `_guess_output_type_by_name()`；后续 R3 拆分时彻底移除 |
| `concurrent_safe=False` 与 `serialized=True` 语义重叠 | 迁移期两者并存，`serialized` 优先；R3 拆分时统一为 `concurrent_safe` |
