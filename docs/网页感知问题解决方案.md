# 🔧 网页感知问题解决方案

## 📊 问题诊断

根据你的日志，主要有三个问题：

### 1. **CNNVD 任务失败**
```
[ToolFailure] step=3 tool=browser_agent expected=成功访问网页并提取到数据 error=repeated action loop detected at step 3
```
**原因**: Agent 不知道如何正确导航到漏洞列表页面，在首页反复点击相同元素

### 2. **广州天气失败**
```
navigation landed on blocked page: 403 Forbidden
```
**原因**:
- 所有尝试的天气网站都返回 403
- Agent 误判为被封禁页面，立即退出
- 没有尝试其他策略

### 3. **北京天气成功的原因**
- Router 有 `deterministic route` 逻辑
- 直接使用了 `tianqi.2345.com`
- 该网站没有反爬限制

## 💡 解决方案

### 方案 A：领域知识注入（推荐）

**原理**: 不是硬编码逻辑，而是将领域知识作为 **prompt 的一部分** 动态注入

**实现**:
1. 创建了 `utils/domain_knowledge.py` - 领域知识库
2. 在 Browser Agent 执行前，检测任务领域
3. 将相关领域知识注入到 prompt 中

**优势**:
- ✅ 保持代码通用性
- ✅ 提供领域专家指导
- ✅ 易于扩展新领域
- ✅ 知识可配置化

**使用方式**:
```python
from utils.domain_knowledge import get_domain_hints

# 在 Browser Agent 的 run 方法中
domain_hints = get_domain_hints(task, start_url)
if domain_hints:
    # 将 domain_hints 添加到 system prompt 或 user prompt 中
    enhanced_prompt = f"{original_prompt}\n\n{domain_hints}"
```

### 方案 B：改进反爬检测逻辑

**问题**: 当前 `_looks_like_blocked_page` 过于严格

**建议修改** (`agents/browser_agent.py` 第 3985 行):

```python
# 当前逻辑（过于严格）
if self._looks_like_blocked_page(current_url, page_title):
    return {
        "success": False,
        "message": f"navigation landed on blocked page: {page_title or current_url}",
        ...
    }

# 改进后（更宽松）
if self._looks_like_blocked_page(current_url, page_title):
    # 不要立即退出，记录警告但继续尝试
    log_warning(f"疑似被封禁页面: {page_title or current_url}, 但继续尝试执行")
    # 可以尝试：
    # 1. 等待更长时间
    # 2. 刷新页面
    # 3. 尝试提取数据（可能是误判）
```

### 方案 C：增强页面感知

**问题**: 页面结构提取不够准确

**建议**:
1. 改进 `utils/page_perceiver.py` 的提取逻辑
2. 增加更多的页面类型识别
3. 提供更详细的页面结构描述给 LLM

### 方案 D：添加重试策略

**问题**: 遇到 403 就放弃，没有尝试其他方法

**建议**:
```python
# 在 web_worker.py 中添加降级策略
if response.status_code == 403:
    # 策略 1: 更换 User-Agent
    # 策略 2: 添加延迟
    # 策略 3: 尝试其他域名
    # 策略 4: 使用 API（如果有）
```

## 🎯 立即可行的改进

### 1. 集成领域知识系统

在 `agents/browser_agent.py` 的 `run` 方法开始处添加：

```python
async def run(self, task: str, start_url: Optional[str] = None, max_steps: int = 8) -> Dict[str, Any]:
    # 添加这几行
    from utils.domain_knowledge import get_domain_hints
    domain_hints = get_domain_hints(task, start_url or "")

    # 将 domain_hints 存储为实例变量，在构建 prompt 时使用
    self._current_domain_hints = domain_hints

    # 原有代码继续...
    tk = self.toolkit
    ...
```

然后在构建 LLM prompt 的地方（比如 `_decide_next_action` 方法）添加：

```python
def _build_action_prompt(self, task, ...):
    prompt_parts = [
        "你是一个网页自动化 Agent...",
        f"任务: {task}",
    ]

    # 添加领域知识
    if hasattr(self, '_current_domain_hints') and self._current_domain_hints:
        prompt_parts.append(self._current_domain_hints)

    return "\n\n".join(prompt_parts)
```

### 2. 放宽反爬检测

修改 `agents/browser_agent.py` 第 3985-3993 行：

```python
if self._looks_like_blocked_page(current_url, page_title):
    # 改为警告而不是立即失败
    log_warning(f"疑似被封禁页面: {page_title or current_url}")
    # 尝试等待和刷新
    await asyncio.sleep(2)
    await tk.reload()
    await self._wait_for_page_ready()
    # 重新检查
    current_url_r = await tk.get_current_url()
    current_url = current_url_r.data or ""
    title_r = await tk.get_title()
    page_title = title_r.data or ""
    # 如果还是被封禁，再返回失败
    if self._looks_like_blocked_page(current_url, page_title):
        return {
            "success": False,
            "message": f"navigation landed on blocked page: {page_title or current_url}",
            ...
        }
```

## 📝 总结

**核心思想**:
- ❌ 不要硬编码特定网站的解析逻辑
- ✅ 将领域知识作为 **prompt 增强** 提供给 LLM
- ✅ 让 LLM 根据领域知识自己决定如何操作
- ✅ 保持代码的通用性和可扩展性

**效果预期**:
- CNNVD 任务: Agent 会知道"首页没有列表，需要点击进入子页面"
- 天气任务: Agent 会知道"可以尝试多个天气网站"
- 新任务: 只需添加领域知识，无需修改代码

这样既保持了系统的通用性，又提供了足够的领域指导！
