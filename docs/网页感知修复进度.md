### 2. 问题1: 提取页面内容元素 - 恢复视觉 ✅

**修改文件：**
- `agents/browser_agent.py:45-54` - 扩展 PageElement 数据结构
- `agents/browser_agent.py:737-893` - 重构 _extract_interactive_elements()
- `agents/browser_agent.py:530-572` - 更新 _format_assessment_elements_for_llm()

**实施内容：**
1. 扩展 `PageElement` 数据结构
   ```python
   @dataclass
   class PageElement:
       # ... 原有字段
       context_before: str = ""  # 元素前面的上下文文本
       context_after: str = ""   # 元素后面的上下文文本
   ```

2. 重构 `_extract_interactive_elements()` 方法
   - 新增 `extractContext()` JavaScript 函数
   - 向前查找最多3个兄弟节点的文本
   - 向后查找最多3个兄弟节点的文本
   - 如果兄弟节点没有上下文，从父元素提取
   - 每个方向最多保留 150 字符

3. 更新 `_format_assessment_elements_for_llm()` 方法
   - 在元素信息中包含 `context_before` 和 `context_after`
   - 格式：`| context: before: xxx | after: xxx`

**预期效果：**
- Agent 能看到按钮/链接周围的文本
- 理解元素的语义上下文
- 减少"盲人摸象"式的瞎猜

**示例：**
```
修复前：
[0] type=link selector=a:nth-of-type(1) info=详情

修复后：
[0] type=link selector=a:nth-of-type(1) info=详情 | context: before: CVE-2024-1234 高危漏洞 | after: 影响范围：全球
```
