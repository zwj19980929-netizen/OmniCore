# P3-2: 多 Agent 协作方案

> 优先级：P3 | 预估工作量：5-7 天 | 依赖：无，但架构复杂，建议 P0/P1 完成后再做

---

## 1. 目标

支持多个 Agent 协同完成复杂任务：Planner 动态调整计划、Executor Agent 独立执行子任务、Agent 间通过消息总线协商。覆盖典型场景：Browser 下载 → Terminal 处理 → File Worker 整理。

**验收标准**：
- 支持 Planner-Executor 分离模式
- 任务间数据依赖通过共享上下文传递（非字符串拼接）
- `graph.py` 支持动态子图（执行阶段可展开为子 DAG）
- 提供 Agent-to-Agent 消息类型

## 3. 架构设计

### 3.1 当前架构（单层 DAG）

```
Router → task_queue = [T1, T2(depends T1), T3(depends T1)]
          ↓
parallel_executor → 批次 1: [T1]
                 → 批次 2: [T2, T3]
                 → 完成
```

### 3.2 目标架构（动态子图）

```
Router → task_queue = [T1(browser), T2(terminal, depends T1), T3(file, depends T2)]
          ↓
parallel_executor → 执行 T1 → 结果写入 shared_context["T1_output"]
          ↓
T1 完成后，T2 从 shared_context 读取 T1 输出作为参数
          ↓
T2 完成后，T3 读取 T2 输出
          ↓
Planner 在任意节点后可"插入"新任务（动态扩展 task_queue）
```

### 3.3 Planner-Executor 分离

```
┌─────────────────────────────────────────────────────┐
│                  PlannerAgent                        │
│  • 生成高层计划（粗粒度任务列表）                      │
│  • 监控执行进度                                       │
│  • 按需插入新任务（动态规划）                          │
│  • 不直接执行，只规划                                 │
└──────────────────────┬──────────────────────────────┘
                       │ 写入 task_queue + 读取执行结果
                       ▼
┌─────────────────────────────────────────────────────┐
│                  ExecutorAgents                      │
│  • Browser / Web / File / System / Terminal         │
│  • 接收 TaskItem，执行，写回结果                     │
│  • 不做规划，只执行                                  │
└─────────────────────────────────────────────────────┘
```

### 4.3 任务间数据传递

修改 `core/task_executor.py` 中的 `_apply_task_outcome()` 函数，在任务完成时写入 `task_outputs`：

```python
def _apply_task_outcome(
    state: OmniCoreState,
    task_idx: int,
    outcome: Dict[str, Any],
) -> None:
    """将任务执行结果写回 state。"""
    # 现有逻辑...
    task = state["task_queue"][task_idx]
    task_id = task.get("task_id", "")

    # 新增：写入类型化输出
    if outcome.get("status") == str(TaskStatus.COMPLETED):
        typed_output = _extract_typed_output(task, outcome)
        if typed_output:
            if "task_outputs" not in state:
                state["task_outputs"] = {}
            state["task_outputs"][task_id] = typed_output


def _extract_typed_output(task: Dict, outcome: Dict) -> Optional[Dict]:
    """从任务结果中提取类型化输出。"""
    result = outcome.get("result", {})
    tool_name = task.get("tool_name", "")

    if "browser" in tool_name or "web" in tool_name:
        return {
            "type": TaskOutputType.TEXT_EXTRACTION,
            "content": result.get("extracted_text", "") or result.get("output", ""),
            "source_url": result.get("url", ""),
        }
    elif "file" in tool_name:
        return {
            "type": TaskOutputType.FILE_DOWNLOAD,
            "file_path": result.get("file_path", ""),
            "file_size": result.get("file_size", 0),
        }
    elif "terminal" in tool_name or "system" in tool_name:
        return {
            "type": TaskOutputType.COMMAND_OUTPUT,
            "stdout": result.get("output", ""),
            "returncode": result.get("returncode", 0),
        }
    return None
```

修改 `core/task_executor.py` 中的参数解析，支持 `$task_id.field` 引用：

```python
def _resolve_task_params(
    params: Dict[str, Any],
    task_outputs: Dict[str, Any],
) -> Dict[str, Any]:
    """
    解析任务参数中的任务输出引用。

    支持格式：
    - "$task_1.file_path" → task_outputs["task_1"]["file_path"]
    - "$task_1.content"   → task_outputs["task_1"]["content"]
    """
    resolved = {}
    for k, v in params.items():
        if isinstance(v, str) and v.startswith("$"):
            ref = v[1:]  # 去掉 $
            parts = ref.split(".", 1)
            ref_task_id = parts[0]
            ref_field = parts[1] if len(parts) > 1 else "content"
            task_out = task_outputs.get(ref_task_id, {})
            resolved[k] = task_out.get(ref_field, v)  # 找不到时保留原值
        else:
            resolved[k] = v
    return resolved
```

### 4.4 动态 Planner（执行中插入任务）

在 `core/graph.py` 中新增一个 `dynamic_replan_node`，在 `parallel_executor` 执行完一批次后触发：

```python
@register_stage("dynamic_replan")
async def _dynamic_replan_node(state: OmniCoreState) -> OmniCoreState:
    """
    动态规划节点：检查已完成的任务输出，决定是否插入新任务。

    触发条件：
    1. 有任务刚完成
    2. 任务输出中有"发现新子任务"的标志
    3. 或 Router 有动态规划指令
    """
    # 检查是否有待插入的动态任务
    additions = state.get("dynamic_task_additions", [])
    if additions:
        # 插入到 task_queue 末尾
        existing_ids = {t["task_id"] for t in state["task_queue"]}
        for new_task in additions:
            if new_task["task_id"] not in existing_ids:
                state["task_queue"].append(new_task)
        state["dynamic_task_additions"] = []

    return state
```

**Router 支持动态规划指令**：允许 Router 在 `tasks` 中声明"条件性子任务"：

在 `core/task_executor.py` 的 `is_task_ready()` 中增加条件检查：

```python
def is_task_ready(task: Dict, task_queue: List[Dict]) -> bool:
    """判断任务是否就绪（依赖已完成 + 条件满足）。"""
    # 现有依赖检查...
    deps = task.get("depends_on", [])
    completed_ids = {t["task_id"] for t in task_queue if t["status"] == "completed"}
    if not all(dep in completed_ids for dep in deps):
        return False

    # 新增：条件检查
    conditional = task.get("conditional")
    if conditional:
        when_expr = conditional.get("when", "")
        # 简单表达式求值（可扩展为完整表达式引擎）
        if not _evaluate_condition(when_expr, task_queue):
            if conditional.get("else_skip"):
                task["status"] = "skipped"  # 条件不满足时跳过
            return False

    return True
```

## 5. 典型场景：Browser 下载 → Terminal 处理 → File 整理

Router 生成的 task_queue：

```json
[
    {
        "task_id": "task_1",
        "tool_name": "browser.navigate_and_download",
        "description": "从目标网站下载报告 PDF",
        "params": {"url": "https://example.com/report", "download": true},
        "depends_on": []
    },
    {
        "task_id": "task_2",
        "tool_name": "terminal_worker",
        "description": "用 pdftotext 提取 PDF 文本",
        "params": {
            "command": "pdftotext $task_1.file_path /tmp/report.txt"
        },
        "depends_on": ["task_1"],
        "conditional": {
            "when": "$task_1.file_path ends_with .pdf",
            "else_skip": false
        }
    },
    {
        "task_id": "task_3",
        "tool_name": "file_worker",
        "description": "读取提取的文本，生成结构化摘要",
        "params": {
            "action": "read",
            "file_path": "/tmp/report.txt",
            "post_process": "summarize"
        },
        "depends_on": ["task_2"]
    }
]
```

执行流：
1. `task_1` 执行，输出 `{"type": "file_download", "file_path": "/tmp/report.pdf"}`
2. `task_2` 就绪，参数 `$task_1.file_path` 解析为 `/tmp/report.pdf`，执行
3. `task_3` 就绪，读取 `/tmp/report.txt` 并汇总

## 7. 风险与注意事项

| 风险 | 缓解措施 |
|------|----------|
| `$ref` 引用解析错误导致任务失败 | 找不到 ref 时保留原始字符串值，任务降级而非崩溃 |
| 动态插入任务导致无限循环 | 限制动态插入最多 10 个任务；检测循环依赖 |
| 状态膨胀（task_outputs 过大） | 每个 task 输出截断 50KB；大文件只存 file_path 引用 |
| 条件表达式安全性 | 不使用 `eval()`；只支持有限的内置操作符 |
| 图结构复杂度增加调试难度 | 完整的 execution_trace 记录每个 Agent 的决策链 |
