# OmniCore 通用 Agent 第一阶段落地说明

日期：2026-03-03

本文是对 [2026-03-03-通用Agent演进路线图.md](/D:/zwj_project/OmniCore/docs/2026-03-03-通用Agent演进路线图.md) 的阶段性落地补充，记录第一阶段已经完成的代码改造范围。

## 已完成

1. 引入任务规范化层
   - 新增 `core/task_planner.py`
   - 将 Router / Replanner 输出统一收敛为运行时任务结构
   - 兼容旧的 `task_type`
   - 补齐 `tool_name`、`risk_level`、`requires_confirmation`

2. 引入确定性权限策略
   - 新增 `core/policy_engine.py`
   - 系统命令默认要求确认
   - 文件写入默认要求确认
   - 交互式浏览器操作按规则要求确认
   - 只读网页抓取与只读浏览器读取默认不要求确认

3. 执行器切换到 Tool Registry 解析路径
   - `TaskExecutor` 在执行前先解析 `tool_name / task_type`
   - 通过 `Tool Registry -> adapter` 做分发
   - 旧任务格式仍保持兼容

4. 重规划重新走权限判断
   - Replanner 新生成的任务会重新计算确认需求
   - 避免重规划结果绕过权限控制

## 当前边界

这一阶段完成的是“运行时已经开始围绕 Tool Registry 和确定性权限策略运转”。

还没有完成的是第二阶段内容：

1. 让 Router 的 prompt 原生输出 `tool_name` 和 `tool_args`
2. 删除执行器中遗留的旧静态分支实现
3. 将权限规则继续下沉到具体工具内部，形成更硬的执行边界
