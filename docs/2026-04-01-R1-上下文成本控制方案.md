# R1: 上下文成本控制方案（Tool Result Budget + History Snip）

> 优先级：P0 | 预估工作量：1-2 天 | 依赖：无 | 被依赖：R7（Session Memory 需复用 snip 逻辑）

---

## 1. 目标

长会话和大工具返回是当前 OmniCore 的两大成本黑洞。一次浏览器抓取可能返回 50KB+ 文本，几轮下来 token 消耗暴增，甚至超出模型上下文窗口导致执行失败。

本方案实现两级成本控制：

- **Tool Result Budget**：工具返回超过阈值时自动截断，保留头尾关键信息
- **History Snip**：历史消息超过上限时裁剪早期消息，保留最近对话完整性

**验收标准**：

- 单次工具返回超过 `TOOL_RESULT_MAX_CHARS` 时自动截断，头尾保留比例可配
- 长会话（>20 轮）的 token 消耗相比无截断降低 40%+
- 截断后不丢失关键信息（URL、文件路径、错误码等保留在头尾）
- 现有所有测试 `pytest tests -q` 不受影响

## 2. 现有代码基础

| 组件 | 文件 | 当前状态 |
|------|------|---------|
| 工具结果回写 | `core/task_executor.py` `_apply_task_outcome()` | 结果原样写回 state，仅对 `task_outputs` 有 50KB 截断 |
| 执行结果合并 | `core/graph.py` `parallel_executor_node()` | 无截断 |
| 历史消息 | `core/state.py` `messages` 字段 | LangGraph `add_messages` 累加器，无上限 |
| LLM 调用 | `core/router.py` `analyze_intent()` / `core/graph.py` replanner/finalize | 每次都把完整 messages 传入，无裁剪 |
| 已有截断 | `core/task_executor.py` `_extract_typed_output()` | 仅对 `task_outputs` 做 50KB 截断，不影响传给 LLM 的消息 |

**关键发现**：

1. `_apply_task_outcome()` 第 3 步将 outcome 全量写入 `state["task_queue"][idx]`，包括 `result` 字段，这是工具原始返回，可能极大
2. `parallel_executor_node()` 完成后将所有 task 结果拼成 `HumanMessage` 追加到 `messages`，这里是成本失控的主要入口
3. `analyze_intent()` 接收 `conversation_history` 参数，直接拼入 user_message，无长度限制
4. `finalize_node()` 和 `replanner_node()` 都从 `state["messages"]` 读取完整历史

## 3. 架构设计

### 3.1 两级截断在执行链路中的位置

```
工具执行完成
    │
    ▼
┌──────────────────────────┐
│  Tool Result Budget      │  ← 第一级：单条工具返回截断
│  truncate_tool_result()  │
│  保留头 60% + 尾 30%     │
│  中间插入截断标记         │
└──────────────────────────┘
    │
    ▼
  写入 state["task_queue"][idx]["result"]
    │
    ▼
  parallel_executor_node 拼接 HumanMessage
    │
    ▼
┌──────────────────────────┐
│  History Snip            │  ← 第二级：历史消息裁剪
│  snip_history()          │
│  保留最近 K 条完整       │
│  更早的只保留摘要        │
└──────────────────────────┘
    │
    ▼
  传入 LLM（router / replanner / finalize）
```

### 3.2 截断策略细节

```
Tool Result Budget:
┌──────────────────────────────────────────────┐
│  原始文本（假设 20000 字符）                   │
│                                              │
│  ┌──────────┬──────────────┬────────────┐    │
│  │ 头部 60% │  [截断标记]  │  尾部 30%  │    │
│  │ 4800 字符 │              │ 2400 字符  │    │
│  └──────────┴──────────────┴────────────┘    │
│                                              │
│  截断标记：                                   │
│  \n[... 已截断 12800 字符，保留头尾关键信息 ...]\n │
│                                              │
│  截断后总长度 ≈ 8000 字符（可配置）            │
└──────────────────────────────────────────────┘

History Snip:
┌──────────────────────────────────────────────┐
│  消息列表（假设 25 条）                        │
│                                              │
│  [ msg_1 ... msg_15 ] → 摘要化（role + 前200字符）│
│  [ msg_16 ... msg_25 ] → 保留完整             │
│                                              │
│  keep_recent = 10，所以 msg_16~25 完整保留    │
│  msg_1~15 只保留摘要                          │
└──────────────────────────────────────────────┘
```

## 4. 详细设计

### 4.1 新建 `utils/context_budget.py`

```python
"""上下文成本控制：工具返回截断 + 历史消息裁剪"""

from config.settings import (
    TOOL_RESULT_MAX_CHARS,
    HISTORY_MAX_MESSAGES,
    HISTORY_KEEP_RECENT,
)


def truncate_tool_result(
    text: str,
    max_chars: int = None,
    head_ratio: float = 0.6,
    tail_ratio: float = 0.3,
) -> str:
    """
    截断工具返回文本，保留头部和尾部关键信息。

    Args:
        text: 工具返回的原始文本
        max_chars: 最大字符数，默认读取 TOOL_RESULT_MAX_CHARS
        head_ratio: 头部保留比例（默认 60%）
        tail_ratio: 尾部保留比例（默认 30%）

    Returns:
        截断后的文本，或原文（如果未超限）
    """
    if max_chars is None:
        max_chars = TOOL_RESULT_MAX_CHARS

    if not isinstance(text, str) or len(text) <= max_chars:
        return text

    truncated_chars = len(text) - max_chars
    head_size = int(max_chars * head_ratio)
    tail_size = int(max_chars * tail_ratio)

    marker = (
        f"\n\n[... 已截断 {truncated_chars} 字符"
        f"（原始 {len(text)} 字符），保留头尾关键信息 ...]\n\n"
    )

    return text[:head_size] + marker + text[-tail_size:]


def truncate_result_in_outcome(outcome: dict) -> dict:
    """
    对 task outcome 中的 result 字段应用截断。
    处理 result 为 str 或 dict（含 data/content/text 键）的情况。
    """
    if not outcome or "result" not in outcome:
        return outcome

    result = outcome["result"]

    if isinstance(result, str):
        outcome["result"] = truncate_tool_result(result)
    elif isinstance(result, dict):
        for key in ("data", "content", "text", "raw_text", "extracted_text"):
            if key in result and isinstance(result[key], str):
                result[key] = truncate_tool_result(result[key])

    return outcome


def snip_history(
    messages: list,
    max_messages: int = None,
    keep_recent: int = None,
    summary_max_chars: int = 200,
) -> list:
    """
    裁剪历史消息列表。保留最近 keep_recent 条完整，更早的只保留摘要。

    Args:
        messages: 完整消息列表
        max_messages: 最大消息数，默认读取 HISTORY_MAX_MESSAGES
        keep_recent: 保留最近完整消息数，默认读取 HISTORY_KEEP_RECENT
        summary_max_chars: 摘要最大字符数

    Returns:
        裁剪后的消息列表
    """
    if max_messages is None:
        max_messages = HISTORY_MAX_MESSAGES
    if keep_recent is None:
        keep_recent = HISTORY_KEEP_RECENT

    if len(messages) <= max_messages:
        return messages

    # 保留最近 keep_recent 条完整
    recent = messages[-keep_recent:]
    older = messages[:-keep_recent]

    # 更早的消息只保留摘要
    snipped = []
    for msg in older:
        snipped.append(_summarize_message(msg, summary_max_chars))

    return snipped + recent


def _summarize_message(msg, max_chars: int = 200) -> dict:
    """将单条消息压缩为摘要形式。"""
    # 兼容 LangGraph 的 BaseMessage 和 plain dict
    if hasattr(msg, "content"):
        role = getattr(msg, "type", "unknown")
        content = msg.content
    elif isinstance(msg, dict):
        role = msg.get("role", msg.get("type", "unknown"))
        content = msg.get("content", "")
    else:
        return msg

    if isinstance(content, list):
        # 多模态消息，取文本部分
        text_parts = [
            p.get("text", "") if isinstance(p, dict) else str(p)
            for p in content
        ]
        content = " ".join(text_parts)

    if isinstance(content, str) and len(content) > max_chars:
        content = content[:max_chars] + f"... [已裁剪 {len(content) - max_chars} 字符]"

    # 返回与原始消息相同格式
    if hasattr(msg, "content"):
        # LangGraph BaseMessage，需要创建同类型新实例
        from langchain_core.messages import HumanMessage, AIMessage, SystemMessage
        msg_map = {"human": HumanMessage, "ai": AIMessage, "system": SystemMessage}
        cls = msg_map.get(role, HumanMessage)
        return cls(content=content)
    else:
        return {**msg, "content": content}
```

```python
# 找到类似这段代码：
#   state["task_queue"][idx]["result"] = outcome.get("result")

# 改为：
from utils.context_budget import truncate_result_in_outcome

def _apply_task_outcome(state, idx, outcome):
    # ... 现有逻辑 ...

    # 截断工具返回结果（在写入 state 之前）
    outcome = truncate_result_in_outcome(outcome)

    # 继续原有的写回逻辑
    task = state["task_queue"][idx]
    task["status"] = outcome.get("status", task["status"])
    task["result"] = outcome.get("result")
    # ... 其余不变 ...
```

## 5. 典型场景

### 场景：用户要求"帮我调研 3 个竞品网站并对比"

**无截断时的 token 消耗**：

```
第 1 轮：router 规划 → 3 个 web_worker 任务（~2000 tokens）
第 2 轮：web_worker_1 返回 → 抓取页面 45000 字符 → 约 15000 tokens
第 3 轮：web_worker_2 返回 → 抓取页面 38000 字符 → 约 12700 tokens
第 4 轮：web_worker_3 返回 → 抓取页面 52000 字符 → 约 17300 tokens
第 5 轮：finalize → 需要把前 4 轮全部消息传入 → 约 47000 tokens

总计 LLM 输入 tokens：~94000（其中 ~45000 是重复传入的工具结果）
```

**启用截断后**：

```
第 1 轮：router 规划 → 不变（~2000 tokens）
第 2 轮：web_worker_1 返回 → 截断为 8000 字符 → 约 2700 tokens
第 3 轮：web_worker_2 返回 → 截断为 8000 字符 → 约 2700 tokens
第 4 轮：web_worker_3 返回 → 截断为 8000 字符 → 约 2700 tokens
第 5 轮：finalize → 历史消息已截断 → 约 12000 tokens

总计 LLM 输入 tokens：~22100（降低 ~76%）
```

## 7. 风险与注意事项

| 风险 | 缓解措施 |
|------|---------|
| 截断后丢失关键信息（如错误码在中间） | 头尾保留比例 60%+30%，关键信息通常在头部（摘要/标题）和尾部（结论/错误）；可按需调整比例 |
| `snip_history` 破坏 LangGraph 消息格式 | `_summarize_message` 兼容 BaseMessage 和 dict 两种格式，返回同类型对象 |
| 裁剪后 replanner 无法看到早期失败细节 | `_apply_task_outcome` 截断的是 result 原文，不影响 `execution_trace` 和 `status` 字段 |
| 配置值选择不当导致截断过激或无效 | 默认值保守（8000 字符约 2700 tokens），用户可通过环境变量调整 |
| 多模态消息（图片+文本）的 snip 兼容性 | `_summarize_message` 对 list 类型 content 做文本提取，非文本部分跳过 |
