# 根目录历史文档整合说明

## 目的

根目录曾长期堆积一批阶段性报告、修复总结、调试说明和临时方案文档。它们内容高度重复，且大多只对应某一次迭代或某一轮排障，继续放在根目录会持续污染项目入口。

这次清理后的规则是：

- 根目录只保留真正的入口文档：`README.md`、`AGENTS.md`
- 历史性、阶段性、排障性 Markdown 不再留在根目录
- 这些文档的主题和用途统一收敛到本文件，具体细节如有需要请直接查 Git 历史

## 当前文档入口

- 项目总入口：`README.md`
- Agent / 协作约束：`AGENTS.md`
- 现有详细设计和方案文档：`docs/` 目录内各专题文档
- 本文件：根目录历史文档的整合索引和去留说明

## 清理后的约束

后续新增文档按下面的规则放置：

- 项目入口级说明：只允许放在根目录 `README.md`
- 临时方案、修复记录、问题分析：放 `docs/`
- 某次迭代总结：优先合并进已有专题文档，不再新增根目录 `*_SUMMARY.md` / `*_REPORT.md`
- 临时调试说明：如果不是长期维护文档，不应进入根目录

## 备注

本次清理以“减少根目录噪音”为目标，不保留这些历史 Markdown 的逐文件副本。若后续需要查看当时的完整原文，请直接从 Git 历史中检索对应文件。
