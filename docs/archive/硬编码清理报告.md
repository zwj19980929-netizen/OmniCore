# 硬编码清理报告

## 📋 概述

根据 CLAUDE.md 的要求："prompt中只能出现**引导思考**型的，不允许出现硬编码的prompt"，我们对整个项目进行了硬编码清理，使系统更加通用和可维护。

## ✅ 已完成的改进

### 1. **agent_critic.py - 批评系统去硬编码**

#### 问题
- 硬编码了 CNNVD 漏洞查询的特定策略建议（第 216-253 行）
- 硬编码了 Google vs DuckDuckGo 的搜索引擎判断

#### 解决方案
- ✅ 将 `suggest_alternative_strategy` 改为使用 LLM 动态生成建议
- ✅ 添加 `llm_client` 参数，让系统根据实际任务描述生成针对性建议
- ✅ 保留通用模板作为 fallback，但不包含特定任务的硬编码
- ✅ 搜索引擎检测改为通用检测，支持所有常见搜索引擎

#### 代码变化
```python
# 之前：硬编码 CNNVD 策略
return """
🎯 **方案 A：直接访问 CNNVD 官网（推荐，别再搜索了！）**
   1. 直接导航到 https://www.cnnvd.org.cn/
"""

# 之后：使用 LLM 动态生成
if llm_client:
    response = llm_client.chat_with_system(
        system_prompt="你是一个毒舌的 AI 批评家...",
        user_message=f"任务已经失败 {failed_attempts} 次了。请分析并提供替代策略...",
        temperature=0.8,
    )
    return response
```

## 🎯 改进效果

### 之前的问题
1. **不通用**：系统为 CNNVD、Hacker News 等特定任务硬编码了逻辑
2. **难维护**：关键词分散在多个文件中，修改困难
3. **不灵活**：无法适应新的任务类型
4. **违反原则**：违反了 CLAUDE.md 的"不允许硬编码 prompt"原则

### 现在的优势
1. **通用性强**：系统可以处理任何类型的网页抓取任务
2. **易维护**：所有关键词集中在配置文件中
3. **动态生成**：使用 LLM 动态生成针对性的建议
4. **符合原则**：遵循"引导思考"而非硬编码的原则
