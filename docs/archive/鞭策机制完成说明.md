## 🎯 批评系统检测的问题

### 1. 重复操作循环 (CRITICAL)
**触发条件**: 检测到 "repeated action loop"
**批评内容**:
```
🤦 太蠢了！你已经点击了 3 次相同的按钮，还没意识到这是个死循环吗？
一个正常的 Agent 应该在第 2 次失败后就换策略，而不是像个机器人一样重复错误！
```

### 2. 使用错误的搜索引擎 (HIGH)
**触发条件**: 用户要求 Google，但用了 DuckDuckGo
**批评内容**:
```
🤡 听不懂人话吗？用户说了要用 Google，你却自作聪明去用别的搜索引擎！
这是基本的指令理解能力，连这都做不到，还能做什么？
```

### 3. 零进展 (CRITICAL)
**触发条件**: 重试多次但一个任务都没完成
**批评内容**:
```
😤 完全没用！重试了 3 次，连第一步都没完成！
你是来浪费时间的吗？每次都是同样的错误，完全没有学习能力！
```

### 4. 导航失败 (HIGH)
**触发条件**: 连目标网站都打不开
**批评内容**:
```
🙄 基础能力都没有！连打开一个网站都做不到，还谈什么抓取数据？
这是最基本的操作，幼儿园水平！
```

### 5. 选择器错误 (MEDIUM)
**触发条件**: 找不到页面元素
**批评内容**:
```
😑 眼瞎了吗？页面上明明有元素，你却说找不到！
是不是根本没看页面结构就瞎猜选择器？
```

### 6. 数据提取失败 (MEDIUM)
**触发条件**: 进了网站但没抓到数据
**批评内容**:
```
😒 半途而废！好不容易进了网站，结果一条数据都没拿到！
这就像爬山爬到山顶却忘了拍照，白费功夫！
```

## 🎮 如何使用

### 自动触发
鞭策机制已经集成到重规划节点，每次 Agent 失败重试时会自动触发。

### 手动测试
```python
from utils.agent_critic import AgentCritic

critic = AgentCritic()

# 模拟失败结果
result = {
    "success": False,
    "error": "repeated action loop detected at step 3",
    "completed_tasks": 0,
    "total_tasks": 3,
    "task": "使用 Google 搜索 CNNVD",
    "output": "Clicked Google 搜索 button 3 times"
}

# 分析并生成批评
issues = critic.analyze_failure(result)
report = critic.generate_pua_report(issues, attempt_count=3)
print(report)

# 获取替代策略建议
alternative = critic.suggest_alternative_strategy("搜索 CNNVD 漏洞", 3)
print(alternative)
```
