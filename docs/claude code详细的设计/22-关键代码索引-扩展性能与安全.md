# 关键代码索引：扩展、性能、安全

这篇覆盖三类平台层能力：

- 扩展性：Tool / Skill / Tool Search / MCP 接入
- 性能：Compact / Session Memory Compact / Tool Result Budget
- 安全：Permission Mode / Bash / PowerShell

## 1. 能力协议层

### `Tool.ts`

- 文件地址：`Tool.ts`
- 关键对象：`Tool` 接口、`buildTool(...)`
- 为什么关键：上层 runtime 面向的不是函数，而是带运行语义的能力对象。
- 关键伪代码：

```ts
type Tool = {
  name
  description()
  inputSchema
  outputSchema
  validateInput?
  checkPermissions?
  isConcurrencySafe?
  isReadOnly?
  shouldDefer?
  searchHint?
  call(input, context)
}

function buildTool(def) {
  fillDefaults()
  attachCommonRuntimeBehavior()
  return toolObject
}
```

### `tools.ts`

- 文件地址：`tools.ts`
- 关键函数：工具注册与裁剪逻辑
- 为什么关键：不同环境、模式、feature gate 下，真正暴露给模型的工具集合不是固定的。
- 关键伪代码：

```ts
function buildToolRegistry(runtimeState) {
  tools = allTools()
  tools = filterByEnvironment(tools)
  tools = filterByFeatureFlags(tools)
  tools = filterByDenyRules(tools)
  tools = filterByMode(simple/repl/remote)
  return tools
}
```

## 2. 工具检索

### `utils/toolSearch.ts`

- 文件地址：`utils/toolSearch.ts`
- 关键函数：`getToolSearchMode()`、`getAutoToolSearchCharThreshold()`、`isToolSearchEnabled()`、`extractDiscoveredToolNames()`、`getDeferredToolsDelta()`
- 为什么关键：工具不是全量常驻 prompt，而是按上下文成本和查询信号动态发现。
- 关键伪代码：

```ts
function getToolSearchMode() {
  return oneOf('standard', 'tst', 'tst-auto')
}

async function isToolSearchEnabled(model, tools, messages) {
  if (mode === 'standard') return false
  if (mode === 'tst') return true
  if (mode === 'tst-auto') {
    chars = calculateDeferredToolDescriptionChars(tools)
    return chars > getAutoToolSearchCharThreshold(model)
  }
}

function getDeferredToolsDelta(tools, messages) {
  discovered = extractDiscoveredToolNames(messages)
  return diffVisibleAndAlreadyDiscoveredTools(tools, discovered)
}
```

### `tools/ToolSearchTool/ToolSearchTool.ts`

- 文件地址：`tools/ToolSearchTool/ToolSearchTool.ts`
- 关键函数：`parseToolName(...)`、`searchToolsWithKeywords(...)`、`ToolSearchTool.call()`
- 为什么关键：把“选工具”从 prompt 常量选择问题变成检索问题。
- 关键伪代码：

```ts
function parseToolName(name) {
  if (name startsWith 'mcp__') {
    return splitIntoServerAndActionParts(name)
  }
  return splitCamelCaseAndUnderscore(name)
}

async function searchToolsWithKeywords(query, deferredTools, allTools, maxResults) {
  if (exactToolNameMatch(query)) return [exactToolName]
  if (mcpPrefixMatch(query)) return matchingMcpTools

  terms = parseRequiredAndOptionalTerms(query)
  candidates = filterDeferredToolsByRequiredTerms(terms)

  score each candidate by:
    exact part match
    partial match
    searchHint match
    description match

  return topScoredNames(maxResults)
}

const ToolSearchTool = buildTool({
  shouldDefer: false,
  isReadOnly: true,
  async call({ query, max_results }, { options: { tools }, getAppState }) {
    deferredTools = tools.filter(isDeferredTool)
    maybeInvalidateCache(deferredTools)

    if (query startsWith 'select:') {
      return directSelectionMode()
    }

    matches = await searchToolsWithKeywords(query, deferredTools, tools, max_results)
    return buildSearchResult(matches, query, deferredTools.length, pendingMcpServers)
  }
})
```

### `tools/SkillTool/SkillTool.ts`

- 文件地址：`tools/SkillTool/SkillTool.ts`
- 关键函数：`getAllCommands()`、`executeForkedSkill(...)`、`SkillTool.call()`
- 为什么关键：skill 不只是被读到 prompt，而是真的可以在 forked context 中运行。
- 关键伪代码：

```ts
async function getAllCommands(context) {
  mcpSkills = context.getAppState().mcp.commands.filter(isMcpSkill)
  localCommands = await getCommands(getProjectRoot())
  return uniqBy([...localCommands, ...mcpSkills], 'name')
}

async function executeForkedSkill(command, commandName, args, context, canUseTool, parentMessage) {
  { modifiedGetAppState, baseAgent, promptMessages, skillContent } =
    await prepareForkedCommandContext(command, args, context)

  agentDefinition = command.effort ? { ...baseAgent, effort: command.effort } : baseAgent

  for await (message of runAgent({
    agentDefinition,
    promptMessages,
    toolUseContext: { ...context, getAppState: modifiedGetAppState },
    canUseTool,
    querySource: 'agent:custom',
    override: { agentId },
  })) {
    collectAgentMessages(message)
    reportSkillProgressIfNeeded(message)
  }

  return { success: true, status: 'forked', agentId, result: extractResultText(...) }
}
```

## 4. 性能与压缩

### `services/compact/autoCompact.ts`

- 文件地址：`services/compact/autoCompact.ts`
- 关键函数：`getEffectiveContextWindowSize()`、`getAutoCompactThreshold()`、`calculateTokenWarningState()`、`shouldAutoCompact()`、`autoCompactIfNeeded()`
- 为什么关键：把上下文窗口当正式资源，决定何时自动 compact。
- 关键伪代码：

```ts
function shouldAutoCompact(messages, model, tracking) {
  if (!autoCompactEnabled) return false
  if (tracking.consecutiveFailures beyond breaker) return false

  tokenUsage = tokenCountWithEstimation(messages)
  threshold = getAutoCompactThreshold(model)
  return tokenUsage >= threshold
}

async function autoCompactIfNeeded(messages, context, cacheSafeParams, querySource, tracking) {
  if (!shouldAutoCompact(...)) return { compactionResult: null }

  smCompact = await trySessionMemoryCompaction(messages)
  if (smCompact) return { compactionResult: smCompact }

  fullCompact = await compactConversation(messages, cacheSafeParams, ...)
  return { compactionResult: fullCompact }
}
```

## 5. 权限模式

### `utils/permissions/permissionSetup.ts`

- 文件地址：`utils/permissions/permissionSetup.ts`
- 关键函数：`removeDangerousPermissions()`、`stripDangerousPermissionsForAutoMode()`、`restoreDangerousPermissions()`、`transitionPermissionMode()`、`prepareContextForPlanMode()`
- 为什么关键：权限模式不是 UI 标签，而是允许规则集合会跟着变化。
- 关键伪代码：

```ts
function stripDangerousPermissionsForAutoMode(context) {
  rules = collectAlwaysAllowRules(context)
  dangerous = findDangerousClassifierPermissions(rules, [])
  stripped = stashDangerousRulesBySource(dangerous)
  return {
    ...removeDangerousPermissions(context, dangerous),
    strippedDangerousRules: stripped,
  }
}

function restoreDangerousPermissions(context) {
  for (source, ruleStrings of context.strippedDangerousRules) {
    context = applyPermissionUpdate(context, {
      type: 'addRules',
      rules: ruleStrings,
      behavior: 'allow',
      destination: source,
    })
  }
  return { ...context, strippedDangerousRules: undefined }
}

function transitionPermissionMode(fromMode, toMode, context) {
  if (fromMode === toMode) return context

  handlePlanModeTransition(fromMode, toMode)
  handleAutoModeTransition(fromMode, toMode)

  if (toMode === 'plan') {
    return prepareContextForPlanMode(context)
  }

  if (toMode === 'auto') {
    context = stripDangerousPermissionsForAutoMode(context)
  } else if (leavingClassifierMode) {
    context = restoreDangerousPermissions(context)
  }

  return maybeClearPrePlanMode(context)
}
```

## 6. Bash 安全

### `tools/BashTool/bashPermissions.ts`

- 文件地址：`tools/BashTool/bashPermissions.ts`
- 关键函数：`getSimpleCommandPrefix()`、`getFirstWordPrefix()`、`stripSafeWrappers()`、`matchingRulesForInput()`、`checkCommandAndSuggestRules()`、`bashToolHasPermission()`
- 为什么关键：shell 安全不是黑名单，而是结构化分析、规则匹配和建议生成。
- 关键伪代码：

```ts
function bashToolHasPermission(command, permissionContext) {
  normalized = stripSafeWrappers(command)
  prefix = getSimpleCommandPrefix(normalized) ?? getFirstWordPrefix(normalized)

  if (checkEarlyExitDeny(normalized)) return deny()
  if (matchesExistingAllowRule(prefix, normalized)) return allow()
  if (checkSemanticsDeny(normalized)) return deny()

  suggestion = checkCommandAndSuggestRules(normalized)
  return askWithSuggestedPermissionRule(suggestion)
}
```

## 7. PowerShell 安全

### `tools/PowerShellTool/pathValidation.ts`

- 文件地址：`tools/PowerShellTool/pathValidation.ts`
- 关键函数：`isDangerousRemovalRawPath()`、`dangerousRemovalDeny()`、`validatePath()`、`checkPathConstraints()`
- 为什么关键：Windows/PowerShell 的路径与 cmdlet 语义被单独处理，不和 Bash 混用。
- 关键伪代码：

```ts
function checkPathConstraints(parsedCommand, sandboxRules) {
  paths = extractPathsFromCommand(parsedCommand)
  for (path of paths) {
    expanded = expandTilde(path)
    if (isDangerousRemovalRawPath(expanded)) {
      return dangerousRemovalDeny(expanded)
    }
    if (!isPathAllowed(expanded, sandboxRules)) {
      return deny()
    }
    if (matchesDenyRule(expanded)) {
      return deny()
    }
  }
  return allow()
}
```
