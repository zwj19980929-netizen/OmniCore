# claude-code-cli 深度分析索引

这组文档不是项目使用说明，而是从“Claude Code 这套 agent runtime 为什么强、强在哪里、怎么做出来的”这个角度，反向拆它的关键工程设计。重点不是功能罗列，而是抓住它和外界很多 agent 系统不一样的核心方法论：

1. 它怎么做意图识别与输入路由。
2. 它怎么做分层记忆与上下文治理。
3. 它怎么把规划做成运行时状态，而不是一句提示词。
4. 它怎么把 agent loop 做成显式执行内核。
5. 它怎么做 Harness Engineering，而不只是“prompt engineering”。
6. 它怎么维护仓库世界模型，从全局去写代码。

## 先给结论

这套代码真正有竞争力的地方，不是“工具多”，也不是“模型强”，而是它把 agent 做成了一套工程化 runtime：

- 意图识别不是一个分类器，而是一条多段式路由链。
- 记忆不是把聊天历史塞回 prompt，而是分成静态项目记忆、长期文件记忆、会话记忆、相关记忆检索、文件状态缓存。
- 规划不是提示词话术，而是 Plan Mode、Todo、Task、Subagent、审批流、后台任务组成的运行时状态机。
- 写代码不是单次补全，而是依赖持续维护的仓库世界模型。
- agent loop 不是“模型自己想”，而是显式的 query loop、tool loop、recovery loop、compact loop。
- Harness Engineering 是一等公民：系统提示词分层、动态信息迁移到 attachment lane、prompt cache 优化、streaming tool execution、fork cache 复用，全都在围绕质量与成本做联合优化。

## 一句话总结

如果要一句话概括这个项目，可以这样说：

> 这不是一个“给大模型接了几个工具”的 CLI，而是一套围绕输入路由、记忆分层、规划状态机、全局仓库世界模型、agent loop、prompt cache 和权限治理构建出来的智能体执行内核。
