# 关键代码索引：意图识别与记忆

这篇不是概念分析，而是“顺着代码看”的索引版。每个条目都按同一种格式写：

- 文件地址
- 关键函数
- 为什么关键
- 关键伪代码

## 1. 查询总入口

### `QueryEngine.ts`

- 文件地址：`QueryEngine.ts`
- 关键函数：`class QueryEngine`、`ask()`、内部的 `processUserInputContext` 组装、`buildSystemInitMessage(...)`、`recordTranscript(...)`
- 为什么关键：这是整条 query 生命周期的主调度器，持有 `mutableMessages`、`readFileState`、`loadedNestedMemoryPaths`、`discoveredSkillNames` 这些会话级事实。
- 关键伪代码：

```ts
class QueryEngine {
  state = {
    mutableMessages,
    readFileState,
    loadedNestedMemoryPaths,
    discoveredSkillNames,
    permissionDenials,
  }

  async *ask(prompt) {
    { defaultSystemPrompt, userContext, systemContext } =
      await fetchSystemPromptParts(...)

    processUserInputContext = {
      messages: mutableMessages,
      readFileState,
      loadedNestedMemoryPaths,
      discoveredSkillNames,
      options: { tools, commands, model, mcpClients },
    }

    result = await processUserInput(prompt, processUserInputContext)
    mutableMessages.push(...result.messages)

    await recordTranscript(mutableMessages) // 先落盘，保证可恢复
    yield buildSystemInitMessage(...)

    for await (message of query(...)) {
      mutableMessages.push(message)
      maybeRecordTranscript(message)
    }
  }
}
```

### `query.ts`

- 文件地址：`query.ts`
- 关键函数：`query()`、`queryLoop()`
- 为什么关键：真正决定每轮送给模型的上下文长什么样，也负责把 memory prefetch、skill prefetch、compact、tool result budget 串进主循环。
- 关键伪代码：

```ts
async function* queryLoop(params) {
  state = {
    messages,
    toolUseContext,
    autoCompactTracking,
    pendingToolUseSummary,
    turnCount,
  }

  using pendingMemoryPrefetch = startRelevantMemoryPrefetch(messages, toolUseContext)

  while (true) {
    pendingSkillPrefetch = startSkillDiscoveryPrefetch(null, messages, toolUseContext)

    messagesForQuery = getMessagesAfterCompactBoundary(messages)
    messagesForQuery = applyToolResultBudget(messagesForQuery, contentReplacementState)
    messagesForQuery = maybeSnip(messagesForQuery)
    messagesForQuery = maybeMicrocompact(messagesForQuery)
    messagesForQuery = maybeAutocompact(messagesForQuery)

    response = sampleModel(messagesForQuery, systemPrompt, userContext, systemContext)
    toolCalls = extractToolCalls(response)
    yield* runTools(toolCalls)
    mergeToolResultsBackIntoState()
  }
}
```

## 2. 输入路由

### `utils/processUserInput/processUserInput.ts`

- 文件地址：`utils/processUserInput/processUserInput.ts`
- 关键函数：`processUserInput()`、`processUserInputBase()`、`applyTruncation()`、`addImageMetadataMessage()`
- 为什么关键：它把一段原始输入拆成命令、附件、普通 prompt、hook 附件，是真正的第一层输入编排器。
- 关键伪代码：

```ts
async function processUserInput(args) {
  if (mode === 'prompt' && input is visible user text) {
    showUserInputImmediately()
  }

  result = await processUserInputBase(...)
  if (!result.shouldQuery) return result

  for await (hookResult of executeUserPromptSubmitHooks(...)) {
    if (hookResult.blockingError) return warningOnly()
    if (hookResult.preventContinuation) return stoppedResult()
    if (hookResult.additionalContexts) appendHookAttachment()
    if (hookResult.message) appendHookMessage()
  }

  return result
}

async function processUserInputBase(...) {
  normalizeInputBlocks()
  normalizePastedImagesAndText()

  if (looksLikeSlashCommand && slashCommandsEnabled) {
    return maybeRunSlashCommand(...)
  }

  if (containsUltraPlanSignal) {
    routeToUltraPlan()
  }

  attachments = skipAttachments ? [] : await getAttachmentMessages(...)
  userPrompt = processTextPrompt(...)

  return {
    messages: [...attachments, userPrompt],
    shouldQuery: true,
  }
}
```

### `utils/attachments.ts`

- 文件地址：`utils/attachments.ts`
- 关键函数：`getAttachments()`、`getAttachmentMessages()`、`processAtMentionedFiles()`、`processAgentMentions()`、`processMcpResourceAttachments()`、`getChangedFiles()`、`getNestedMemoryAttachments()`、`getRelevantMemoryAttachments()`、`startRelevantMemoryPrefetch()`
- 为什么关键：这是隐式意图增强器。它会把用户输入和当前运行时缺失的上下文补成 attachment 消息。
- 关键伪代码：

```ts
async function getAttachments(input, toolUseContext, ideSelection, queuedCommands, messages) {
  if (attachmentsDisabled) return getQueuedCommandAttachments(queuedCommands)

  context = withTimeoutAbortController(toolUseContext, 1000ms)
  isMainThread = !toolUseContext.agentId

  userInputAttachments = [
    processAtMentionedFiles(input),
    processMcpResourceAttachments(input),
    processAgentMentions(input),
    maybeTurnZeroSkillDiscovery(input),
  ]
  await Promise.all(userInputAttachments)

  sharedAttachments = [
    queued_commands,
    date_change,
    deferred_tools_delta,
    agent_listing_delta,
    changed_files,
    nested_memory,
    dynamic_skill,
    skill_listing,
    plan_mode,
    plan_mode_exit,
    todo_or_task_reminders,
    teammate_mailbox,
    agent_pending_messages,
    critical_system_reminder,
    compaction_reminder,
  ]

  if (isMainThread) {
    add ide_selection + opened_file + diagnostics + lsp_diagnostics
  }

  return flatten(await Promise.all(sharedAttachments))
}
```

## 3. 仓库背景与静态项目记忆

### `context.ts`

- 文件地址：`context.ts`
- 关键函数：`getGitStatus()`、`getSystemContext()`、`getUserContext()`
- 为什么关键：所有 query 开始前都会拿到一份 git 快照和 `CLAUDE.md` 内容，这决定了模型从一开始就不是“盲写代码”。
- 关键伪代码：

```ts
async function getGitStatus() {
  if (!isGitRepo()) return null

  [branch, mainBranch, status, log, userName] = await Promise.all([
    git branch,
    git default branch,
    git status --short,
    git log -n 5,
    git config user.name,
  ])

  return formatGitSnapshot(branch, mainBranch, status, log, userName)
}

async function getSystemContext() {
  gitStatus = shouldIncludeGitInstructions() ? await getGitStatus() : null
  injection = maybeGetSystemPromptInjection()
  return { gitStatus, cacheBreaker: injection }
}

async function getUserContext() {
  claudeMd = shouldDisableClaudeMd ? null : getClaudeMds(await getMemoryFiles())
  return { claudeMd, currentDate }
}
```

- 关键函数：`getMemoryFiles()`、`processMemoryFile()`、`processMdRules()`、`processConditionedMdRules()`、`getClaudeMds()`
- 为什么关键：这一层负责扫描、解析、拼装多层 `CLAUDE.md` / memory 文件，把项目规则、用户规则、条件规则整合成静态上下文。
- 关键伪代码：

async function getMemoryFiles() {
  files = walkManagedUserProjectAndCwdHierarchy()

async function processMemoryFile(filePath) {
  raw = safelyReadMemoryFileAsync(filePath)

function getClaudeMds(files) {
  return files
    .sort(byPrecedenceAndDirectoryDepth)
    .map(renderMemoryFileIntoPrompt)
    .join('\n\n')

## 4. 长期记忆

### `memdir/memdir.ts`

- 文件地址：`memdir/memdir.ts`
- 关键函数：`truncateEntrypointContent()`、`buildMemoryLines()`、`buildMemoryPrompt()`、`loadMemoryPrompt()`
- 为什么关键：真正定义了长期 memory 的数据模型和使用说明，而且明确限制 `MEMORY.md` 只是索引，不是把所有记忆写死一个文件。
- 关键伪代码：

```ts
function buildMemoryLines(displayName, memoryDir) {
  return [
    `You have a persistent file-based memory system at ${memoryDir}`,
    'save only useful future memories',
    'do not use memory for current-task steps',
    'use one file per memory with frontmatter',
    'MEMORY.md is only an index',
  ]
}

function buildMemoryPrompt({ displayName, memoryDir }) {
  entrypointContent = readFileIfExists(memoryDir + '/MEMORY.md')
  lines = buildMemoryLines(displayName, memoryDir)
  lines += truncateEntrypointContent(entrypointContent)
  return lines.join('\n')
}

async function loadMemoryPrompt() {
  ensureMemoryDirExists()
  return buildMemoryPrompt(...)
}
```

## 5. 会话工作记忆

### `services/SessionMemory/sessionMemory.ts`

- 文件地址：`services/SessionMemory/sessionMemory.ts`
- 关键函数：`shouldExtractMemory()`、`setupSessionMemoryFile()`、`extractSessionMemory()`、`createMemoryFileCanUseTool()`
- 为什么关键：这层保存的是“当前长任务状态”，不是长期偏好。它决定什么时候该把当前任务浓缩成结构化 session memory。
- 关键伪代码：

```ts
function shouldExtractMemory(messages) {
  tokenCount = tokenCountWithEstimation(messages)
  if (!sessionMemoryInitialized && tokenCount < initThreshold) return false

  hasMetTokenThreshold = tokenCountGrowthSinceLastExtract >= updateThreshold
  toolCallsSinceLast = countToolCallsSince(messages, lastMemoryMessageUuid)
  hasMetToolCallThreshold = toolCallsSinceLast >= toolCallsBetweenUpdates
  lastTurnHasToolCalls = hasToolCallsInLastAssistantTurn(messages)

  return (
    (hasMetTokenThreshold && hasMetToolCallThreshold) ||
    (hasMetTokenThreshold && !lastTurnHasToolCalls)
  )
}

async function extractSessionMemory(context) {
  if (querySource !== 'repl_main_thread') return
  if (!shouldExtractMemory(messages)) return

  setupContext = createSubagentContext(toolUseContext)
  { memoryPath, currentMemory } = await setupSessionMemoryFile(setupContext)
  updatePrompt = await buildSessionMemoryUpdatePrompt(currentMemory, memoryPath)

  await runForkedAgent({
    promptMessages: [createUserMessage({ content: updatePrompt })],
    canUseTool: createMemoryFileCanUseTool(memoryPath),
    querySource: 'session_memory',
  })
}
```

### `services/compact/sessionMemoryCompact.ts`

- 文件地址：`services/compact/sessionMemoryCompact.ts`
- 关键函数：`adjustIndexToPreserveAPIInvariants()`、`calculateMessagesToKeepIndex()`、`shouldUseSessionMemoryCompaction()`、`trySessionMemoryCompaction()`
- 为什么关键：长会话 compact 时，优先保结构化 session memory，而不是粗暴把所有历史摘要掉。
- 关键伪代码：

```ts
function calculateMessagesToKeepIndex(messages, lastSummarizedIndex) {
  start = lastSummarizedIndex + 1
  while (notEnoughTokensOrTextBlocks && start > compactBoundaryFloor) {
    start--
  }
  return adjustIndexToPreserveAPIInvariants(messages, start)
}

async function trySessionMemoryCompaction(messages) {
  if (!shouldUseSessionMemoryCompaction()) return null
  await waitForSessionMemoryExtraction()
  sessionMemory = await getSessionMemoryContent()
  if (!sessionMemory || isSessionMemoryEmpty(sessionMemory)) return null
  return createCompactionResultFromSessionMemory(...)
}
```
