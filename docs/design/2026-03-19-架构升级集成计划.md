def route_node(state):

### 任务 2: 消息总线集成 (方向 4)

**难度**: 中 | **风险**: 中 | **必须保持 shared_memory 向后兼容**

**目标**: graph.py 的各 stage node 逐步用 `MessageBus.publish/query` 替代 `shared_memory[key]` 直接读写。

**核心策略**: 双写模式 — 写入时同时写 MessageBus 和 shared_memory，读取时优先从 MessageBus 读。

**文件**: `core/message_bus.py` (已完成，不需改动)

**需要修改的文件**:

#### 2.1 `core/graph.py` — 创建辅助函数

在文件顶部添加:
```python
from core.message_bus import MessageBus, MSG_DIRECT_ANSWER, MSG_REPLAN_HISTORY, ...

def _get_bus(state: OmniCoreState) -> MessageBus:
    """Get or create MessageBus from state."""
    bus_data = state.get("message_bus", [])
    return MessageBus.from_dict(bus_data) if bus_data else MessageBus()

def _save_bus(state: OmniCoreState, bus: MessageBus):
    """Save MessageBus back to state."""
    state["message_bus"] = bus.to_dict()
```

#### 2.2 `core/graph.py` — route_node 改写

找到 route_node 里所有 `state["shared_memory"]["router_direct_answer"] = ...` 这类写入，改为:

```python
bus = _get_bus(state)
bus.publish("router", "finalize", MSG_DIRECT_ANSWER, {"answer": direct_answer})
_save_bus(state, bus)
# 同时保留: state["shared_memory"]["router_direct_answer"] = direct_answer  # 向后兼容
```

#### 2.3 逐个 stage 替换

用 `grep 'shared_memory\[' core/graph.py` 找到所有读写点。以下是已知的关键 key 和对应的 stage:

| shared_memory key | 写入 stage | 读取 stage | 对应 message_type |
|---|---|---|---|
| `router_direct_answer` | router | finalize | `MSG_DIRECT_ANSWER` |
| `router_high_risk_reason` | router | human_confirm | `MSG_HIGH_RISK_REASON` |
| `_final_answer_instructions` | router | finalize | `MSG_FINAL_INSTRUCTIONS` |
| `_replan_history` | replanner | replanner (自读) | `MSG_REPLAN_HISTORY` |
| `_approved_actions` | human_confirm | executor | `MSG_APPROVED_ACTIONS` |
| `_resume_after_stage` | runtime | graph entry | `MSG_RESUME_STAGE` |
| `current_time_context` | runtime | router/finalize | `MSG_CONTEXT` |
| `user_location_preference` | runtime | router | `MSG_USER_PREFERENCES` |

每个 key 的迁移步骤相同：写入时 `bus.publish()`，读取时 `bus.get_latest()`，同时保留旧 `shared_memory` 写入。

#### 2.4 最终一致性

当所有 key 都迁移完毕后，可以在 finalize_node 末尾加一个断言:
```python
# Verify bus and shared_memory are in sync
bus = _get_bus(state)
legacy = bus.to_shared_memory()
for k, v in legacy.items():
    assert state["shared_memory"].get(k) is not None, f"Bus/shared_memory desync: {k}"
```

#### 3.2 `core/tool_adapters.py` — WorkerPool 改为 factory 模式

当前 WorkerPool 有硬编码属性 `_web_worker`, `_file_worker`, `_system_worker` 等。改为:

```python
class WorkerPool:
    def __init__(self):
        self._workers: Dict[str, Any] = {}
        self._registry = get_agent_registry()

    def get_worker(self, adapter_name: str):
        """Factory method: lazy-load worker by adapter name."""
        if adapter_name not in self._workers:
            self._workers[adapter_name] = self._create_worker(adapter_name)
        return self._workers[adapter_name]

    def _create_worker(self, adapter_name: str):
        """Create worker instance. Import path inferred from adapter name."""
        if adapter_name == "web_worker":
            from agents.web_worker import WebWorker
            return WebWorker()
        elif adapter_name == "browser_agent":
            from agents.browser_agent import BrowserAgent
            # browser_agent 需要 LLM client，特殊处理
            return None  # 由 adapter 自行创建
        elif adapter_name == "file_worker":
            from agents.file_worker import FileWorker
            return FileWorker()
        elif adapter_name == "system_worker":
            from agents.system_worker import SystemWorker
            return SystemWorker()
        else:
            # 插件 agent —— 从 agent_registry 获取信息动态加载
            agent_def = self._registry.get(adapter_name)
            if agent_def and hasattr(agent_def, 'module_path'):
                # 未来: importlib.import_module(agent_def.module_path)
                pass
            raise ValueError(f"Unknown adapter: {adapter_name}")
```

**注意**: 保留旧的 `@property` 方法作为向后兼容别名，内部委托给 `get_worker()`:
```python
@property
def web_worker(self):
    return self.get_worker("web_worker")
```

#### 3.3 `core/constants.py` — 让实际代码使用注册表

### 任务 5: 可编排图集成 (方向 1)

**难度**: 高 | **风险**: 高 | **建议最后做**

**目标**: `build_execution_plan()` 的 skip_condition 在 conditional edge 函数中被实际评估。

**文件**: `core/stage_registry.py` (已完成), `core/graph.py` (已部分完成)

#### 5.1 问题诊断

当前 `build_graph_from_registry()` 虽然用了 StageRegistry 来添加 node，但 conditional edge 函数仍然是**硬编码的旧函数**:
```python
# 当前状态 (graph.py 里的 build_graph_from_registry):
graph.add_conditional_edges("router", should_continue_after_route, {...})
# ↑ 这些 should_continue_after_route 等函数仍是手写的 if-else
```

需要让这些 routing 函数**查询 StageRegistry 的 skip_condition**。

#### 5.2 修改 conditional edge 函数

以 `after_validator` 为例，当前逻辑:
```python
def after_validator(state):
    if state.get("validator_passed"):
        return "critic"
    else:
        return "replanner"
```

改为:
```python
def after_validator(state):
    registry = StageRegistry.get_instance()
    plan = registry.build_execution_plan(state)

    # 在 plan 里找 validator 之后的下一个 stage
    try:
        idx = plan.index("validator")
        next_stage = plan[idx + 1] if idx + 1 < len(plan) else "finalize"
    except ValueError:
        next_stage = "finalize"
    return next_stage
```

#### 5.3 难点：循环结构

executor ↔ replanner 的循环不适合简单的 "plan 里找下一个" 逻辑。需要特殊处理:

- `after_parallel_executor`: 如果还有 pending tasks → 回到自己(self-loop)，否则 → plan 里的下一个
- `should_retry_or_finish`: 如果 critic 不通过且 replan_count < 3 → replanner，否则 → finalize

**建议**: 对这两个循环 edge 保留硬编码逻辑，其他 edge 走 plan 驱动。纯粹主义不如实用主义。

#### 5.4 Router 输出 stages (可选，更远期)

让 Router 在 JSON 输出里包含 `"stages": ["executor", "validator", "critic", "finalize"]`，然后 `create_initial_state()` 把它写入 `state["execution_plan"]`。`build_execution_plan()` 优先用 state 里的 plan，没有时再走默认 skip_condition 评估。

这一步可以后做，先确保 skip_condition 评估本身能工作。
