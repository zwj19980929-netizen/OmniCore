# 文件生成能力优化方案

**日期：** 2026-03-31
**状态：** 规划中
**涉及文件：** `agents/file_worker.py`, `core/tool_registry.py`, `prompts/router_system.txt`

---

## 一、现状诊断

OmniCore 的文件生成能力集中在 `agents/file_worker.py` 的 `FileWorker` 类中，当前支持：

| 能力 | 现状 |
|------|------|
| 格式支持 | txt / xlsx / csv / markdown / html |
| 数据来源 | shared_memory 的 task_id 映射，支持多源合并 |
| 写入模式 | 仅支持覆盖写入（create-or-overwrite） |
| 内容来源 | 外部任务结果数据（web scraping 等）或 params.content 直接传入 |
| 格式生成 | 硬编码固定模板（HTML 无法自定义，Markdown 仅支持表格） |
| 代码/配置文件 | 不支持（须绕道 terminal_worker） |
| 大文件 | 全量内存加载，无流式写入 |
| 文件追加 | 不支持 |

### 核心痛点

1. **"生成文档"能力弱**：只能把结构化数据序列化成文件，不能基于语义内容生成有逻辑的文档（报告、方案、邮件草稿、README 等）。
2. **格式固定**：HTML/Markdown 输出格式写死，无法按需定制章节、样式、内容结构。
3. **无追加/合并**：无法增量更新已有文件，每次必须全量重写。
4. **代码生成缺口**：生成 .py/.json/.yaml/.js 等代码/配置文件没有专门路径。
5. **大数据导出风险**：全量内存操作，10 万行 CSV 可能 OOM。
6. **Artifact 元数据不足**：Artifact 目录无法记录文件的 schema、行数、字段，无法做语义搜索。

## 二、优化方向与优先级

### P0 — 核心文档生成能力（高价值、阻塞场景）

#### P0-1：LLM 驱动的文档内容生成

**问题：** 当前 file_worker 只能把已有数据序列化，不能凭借语义理解生成新内容。
**方案：** 增加 `action="generate"` 模式，接收 `topic`/`outline`/`style` 参数，调用 LLM 生成文档正文，再写入文件。

```
params:
  action: "generate"
  file_path: "~/Desktop/项目方案.md"
  topic: "OmniCore P2-3 能力扩展方案"
  style: "技术文档"        # 可选: 技术文档 / 邮件 / 周报 / README
  outline: ["背景", "目标", "方案", "风险"]  # 可选: 自定义章节
  format: "markdown"
```

**实现要点：**
- 新增 `prompts/file_generate.txt` 模板，参数化 topic/outline/style
- `FileWorker._generate_content()` 方法调用 LiteLLM 生成正文
- 支持流式输出（streaming=True），写入时实时落盘

**收益：** 解锁"帮我写一份报告/方案/周报/邮件"等高频场景。

---

#### P0-2：Jinja2 模板引擎支持

**问题：** 硬编码的 HTML/Markdown 模板无法复用和定制。
**方案：** 引入 Jinja2，支持用户在 `templates/` 目录下放置自定义模板。

```
params:
  action: "write"
  template: "weekly_report.html.j2"   # templates/ 目录下的模板文件
  data_source: "task_1_data"
  file_path: "~/Desktop/周报.html"
```

**实现要点：**
- `templates/` 目录，内置 `report_table.html.j2`、`report_table.md.j2`
- `FileWorker._render_template()` 方法，接收 template 名称 + 数据字典
- 模板变量：`{{ title }}`、`{{ rows }}`、`{{ generated_at }}`、`{{ metadata }}`

**收益：** 一次写好模板，多次复用；样式可随意定制。

---

### P1 — 写入模式扩展（中等价值、常见需求）

#### P1-1：追加模式（append）

**问题：** 无法增量更新日志、日记、运行记录等文件。
**方案：** `action="append"`，在文件末尾追加内容，自动处理换行符。

```
params:
  action: "append"
  file_path: "~/Desktop/日志.txt"
  content: "2026-03-31 任务完成：爬取了 50 条数据"
```

**实现要点：**
- `open(path, "a", encoding="utf-8")` 写入
- XLSX 追加：`openpyxl.load_workbook()` + append rows
- CSV 追加：`open(path, "a", newline="", encoding="utf-8-sig")`

---

#### P1-2：代码/配置文件生成专用模式

**问题：** 生成 .py/.json/.yaml/.js 等文件时缺乏语法验证和格式化。
**方案：** `action="generate"` 时，根据 `format` 自动做后处理：

| format | 后处理 |
|--------|--------|
| python | `ast.parse()` 语法检查 + `black` 格式化（可选） |
| json | `json.loads()` 验证 + `json.dumps(indent=2)` 格式化 |
| yaml | `yaml.safe_load()` 验证 |
| toml | `tomllib.loads()` 验证（Python 3.11+） |

**Router 提示词扩展：**
```
生成代码/配置文件 → file_worker，action=generate，format=python/json/yaml
```

---

#### P1-3：Artifact 元数据增强

**问题：** Artifact 目录只记录 path/name，无法按"包含销售数据的 xlsx"这类语义检索。
**方案：** 写入成功后，把以下字段加入 artifact preview：

```json
{
  "preview": "rows=1024, cols=['商品','价格','库存'], format=xlsx, size=48KB"
}
```

**实现要点：**
- `FileWorker` 成功后收集：行数、列名、文件大小、格式
- 拼成结构化 preview 字符串存入 artifact catalog
- `artifact_store.search_artifacts()` 可按此文本全文搜索

---

### P2 — 健壮性与大文件支持（长尾改善）

#### P2-1：流式写入大文件

**问题：** 100 万行 CSV 全量加载会 OOM。
**方案：** `data_source` 对应数据为大列表时，分批写入：

```python
# CSV 流式写入
with open(path, "w", newline="", encoding="utf-8-sig") as f:
    writer = csv.DictWriter(f, fieldnames=columns)
    writer.writeheader()
    for chunk in iter_chunks(data, size=10_000):
        writer.writerows(chunk)
```

**触发条件：** `len(data_items) > 50_000` 时自动切换流式模式。

---

#### P2-2：列过滤与字段保护

**问题：** 数据中可能含有敏感字段（API Key、内部 ID），目前全量写出。
**方案：** 新增 `columns` 和 `exclude_columns` 参数：

```
params:
  columns: ["商品名", "价格", "库存"]         # 白名单
  exclude_columns: ["internal_id", "api_key"] # 黑名单
```

---

#### P2-3：格式转换工具

**问题：** 写入 CSV 后用户想要 XLSX，需重新跑任务。
**方案：** `action="convert"`，读取已有文件，转换格式后写出：

```
params:
  action: "convert"
  source_path: "~/Desktop/数据.csv"
  target_path: "~/Desktop/数据.xlsx"
```

---

#### P2-4：压缩打包

**问题：** 多个相关文件无法打包。
**方案：** `action="archive"`：

```
params:
  action: "archive"
  sources: ["~/Desktop/报告.xlsx", "~/Desktop/日志.txt"]
  target_path: "~/Desktop/报告包.zip"
```
