# 文本截断问题分析报告

## 🔴 严重问题：影响网页理解的关键截断

### 1. **agents/web_worker.py:1362** ⚠️ 最严重
```python
if len(html) > 15000:
    html = html[:15000] + "\n... (truncated)"
```
**影响：** 页面分析时，HTML 被截断到 15000 字符，导致大模型无法看到完整页面结构。

**场景：**
- 复杂网页（如 CNNVD）的 HTML 通常超过 15000 字符
- 截断后可能丢失关键的表单、按钮、数据区域
- 导致大模型无法正确定位元素和理解页面结构

---

### 2. **agents/web_worker_singleflight.py:214** ⚠️ 同样严重
```python
if len(html) > 15000:
    html = html[:15000] + "\n... (truncated)"
```
**影响：** 与上面相同，异步版本的 web_worker 也有同样的问题。

---

### 3. **core/tool_adapters.py:609, 619** ⚠️ 中等严重
```python
"content": raw_text[:2000],  # 第 609 行
"content": body[:2000],      # 第 619 行
```
**影响：** HTTP 请求返回的内容被截断到 2000 字符。

**场景：**
- API 返回的 JSON 数据可能被截断
- 网页内容被截断，影响后续分析

---

### 4. **memory/entity_extractor.py:67** ⚠️ 中等影响
```python

```
**影响：** 实体提取时只看前 2000 字符，可能遗漏重要信息。

## 🔧 建议修复方案

### 方案 1：大幅提升截断限制（推荐）
```python
# web_worker.py 和 web_worker_singleflight.py
if len(html) > 100000:  # 从 15000 提升到 100000
    html = html[:100000] + "\n... (truncated)"

# tool_adapters.py
"content": raw_text[:20000],  # 从 2000 提升到 20000

# entity_extractor.py
user_message=f"请从以下文本中提取实体：\n\n{text[:10000]}",  # 从 2000 提升到 10000
```

**优点：**
- 简单直接
- 保留更多上下文
- 现在 max_tokens=65535，模型可以处理更长的输入

**缺点：**
- 可能增加 token 消耗
- 对于超大页面仍可能截断

### 方案 2：智能截断（更优）
```python
def smart_truncate_html(html: str, max_length: int = 100000) -> str:
    """智能截断 HTML，优先保留关键部分"""
    if len(html) <= max_length:
        return html

    # 1. 提取 <head> 部分（包含 meta 信息）
    head_match = re.search(r'<head[^>]*>.*?</head>', html, re.DOTALL | re.IGNORECASE)
    head = head_match.group(0) if head_match else ""

    # 2. 提取 <body> 部分
    body_match = re.search(r'<body[^>]*>(.*?)</body>', html, re.DOTALL | re.IGNORECASE)
    body = body_match.group(1) if body_match else html

    # 3. 移除不重要的部分
    # 移除 <script> 标签
    body = re.sub(r'<script[^>]*>.*?</script>', '', body, flags=re.DOTALL | re.IGNORECASE)
    # 移除 <style> 标签
    body = re.sub(r'<style[^>]*>.*?</style>', '', body, flags=re.DOTALL | re.IGNORECASE)
    # 移除注释
    body = re.sub(r'<!--.*?-->', '', body, flags=re.DOTALL)

    # 4. 如果还是太长，截断 body
    if len(head) + len(body) > max_length:
        body = body[:max_length - len(head) - 100] + "\n... (truncated)"

    return f"{head}<body>{body}</body>"
```

**优点：**
- 保留关键结构
- 移除无用内容（script、style）
- 更高效利用 token

